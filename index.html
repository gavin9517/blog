css中font smoothing属性用法

今天访问larvel官网的时候觉得字体看起来很舒服。查看使用字体时发现了-webkit-font-smoothing这个属性。查询文档得知这个属性可以让MacOS字体更好看(平滑)。
注意的是该属性只能在Mac OS X 平台下生效。

-webkit-font-smoothing主要用到有两个属性subpixel-antialiased和antialiased。
subpixel-antialiased是次级像素平滑。
antialiased是灰度平滑。
只要不是none在MacOS下看到都蛮舒服的。对于非正式项目可以用来试试。

有意思的小知识-费茨法则

来源于维基百科: 费茨法则.

概念

费茨法则 是一个人机互动以及人体工程学中人类活动的模型，它预测了快速移动到目标区域所需的时间是目标区域的距离和目标区域的大小的函数。费茨法则多用于表现 指、点 这个动作的概念模型，无论是用手或者手指进行物理接触，抑或是在电脑屏幕上用假想的设备（例如鼠标）进行虚拟的触碰。 该法则由 Paul Fitts 于 1954 年提出。

模型

费茨法则可以用多种不同的公式进行表达。比较通用的是用于一维运动的 Shannon 公式（由MacKenzie，约克大学教授提出，并因为其与香农定理的相似而命名）。

这里:
T 是完成动作的平均时间。（传统的命名方式，会写成MT ，表示运动时间 Movement Time）
a 代表装置（拦截）开始/结束的时间，b 表示该装置本身的速度（斜率）。这些常数可以以测得数据进行直线近似的方式通过实验取得。
D 是起始位置到目标中心的距离。（传统上来说，研究者多用 A 来表示，意味着运动的振幅 Amplitude）
W 是目标区域在运动维向上的宽度。因为运动的最终点必须落在目标中心 ±W⁄2 以内，所以 W 也可以被认为是被允许的最终位置的容差。
从这个等式我们可以看出，小并且/或者远的目标，需要更长的时间才能得以准确到达，这便是权衡指的动作的 速度的准确性。

magento2 composer 安装 mailgun 模块

magento2的开发中需要用到发信功能,出于性能考虑,最终选择使用 mailgun 的服务来实现发信功能。首先去搜索下github有没有相关的开源程序。很快找到了 Magento 2 Mailgun Module 。

按照说明很简单，只需要 composer require 就可以了。只是这次比较特殊的是需要用户名和密码。之前没遇到过记录下相关操作。

账户名和密码

刚开始一直以为是 magento2 官网注册账户和密码。输入账户名密码一直不正确。这里的账户和密码是 connect 的 public key 、 private key。

访问 https://www.magentocommerce.com/magento-connect/ 登录账户。进入我的账户(My Account)。
点击 Connect 标签。

输入 key 的名称，点击 Generate Key 生成一个 key 。


提交成功之后，public key 就是用户名，private key 就是密码。

设置composer，自动登录

有了账户名和密码，没次输入很麻烦的。可以设置 auth.json 文件让 composer 自动登录。

用文本编辑器打开 ~/.composer/auth.json 文件。参照以下格式配置。

{
    "bitbucket-oauth": {},
    "github-oauth": {},
    "gitlab-oauth": {},
    "http-basic": {
        "repo.magento.com": {
            "username":"<public key>",
            "password":"<private key>"
        }
    }
}

保存成功之后就不用每次都去输入一长串的用户和密码了。
最后，执行 composer require bogardo/mailgun-magento2 命令自动安装成功。也不用每次输入密码了。

magento2 "Unable to retrieve deployment version of static files from the file system."解决

在magento2改为生产模式的时候遇到了一个报错提示。”Command returned non-zero exit code: /usr/local/php7/bin/php -f /xxx/bin/magento setup:static-content:deploy en_US”。应该是部署静态文件出错了。

打开日志文件’system.log’,查看报错信息。

[UnexpectedValueException]
  Unable to retrieve deployment version of static files from the file system.

提示，找不到版本信息。这里应该是magento无法找到/pub/static/deployed_version.txt下文件所致。临时解决办法就是创建一个deployed_version.txt空文件在/pub/static/目录下就可以解决了。

配置ssh key登录github

创建SSH Key

  1. 打开终端

  2. 输入以下命令 

    $ ssh-keygen -t rsa -b 4096 -C "your_email@example.com"

    根据输入的邮箱创建一个ssh key,填入邮箱之后会出现以下的提示信息。这里我们需要设置一个自定义名词。例如：id_rsa_work

    $ Enter a file in which to save the key (/Users/you/.ssh/id_rsa): [Press enter]

    执行完毕之后会生成 id_rsa_work 和 id_rsa_work.pub 两个文件。

添加生成的SSH Key到ssh-agent中

执行以下命令添加ssh key到到 ssh-agent 中。

$ ssh-add ~/.ssh/id_rsa_work

这里，id_rsa_work就是上面自定义的证书名词。

配置ssh

打开 ~/.ssh/config 文件(如果没有则创建一个)。编辑配置文件，使得能够自动识别该用哪个证书。注意缩进

Host work
    HostName github.com
    User git
    IdentityFile ~/.ssh/id_rsa

Host github.com
    HostName github.com
    User git
    IdentityFile ~/.ssh/id_rsa_person

部署到 github 上

  1. 点击顶部的 Setting 链接。
  2. 在账户左边的侧边,点击 SSH and GPG keys
  3. 点击 New SSH key.这里别点错了。
  4. 给 SSH KEY 取个名字.然后,复制文件 id_rsa_work.pub 的内容到输入框，点击add ssh key保存就好了。

添加源

最后添加源就可以了。

git remote add origin git@github.com:gavin9517/gavin9517.github.com.git
