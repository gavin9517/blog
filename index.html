magento2 composer 安装 mailgun 模块

magento2的开发中需要用到发信功能,出于性能考虑,最终选择使用 mailgun 的服务来实现发信功能。首先去搜索下github有没有相关的开源程序。很快找到了 Magento 2 Mailgun Module 。

按照说明很简单，只需要 composer require 就可以了。只是这次比较特殊的是需要用户名和密码。之前没遇到过记录下相关操作。

账户名和密码

刚开始一直以为是 magento2 官网注册账户和密码。输入账户名密码一直不正确。这里的账户和密码是 connect 的 public key 、 private key。

访问 https://www.magentocommerce.com/magento-connect/ 登录账户。进入我的账户(My Account)。
点击 Connect 标签。

输入 key 的名称，点击 Generate Key 生成一个 key 。


提交成功之后，public key 就是用户名，private key 就是密码。

设置composer，自动登录

有了账户名和密码，没次输入很麻烦的。可以设置 auth.json 文件让 composer 自动登录。

用文本编辑器打开 ~/.composer/auth.json 文件。参照以下格式配置。

{
    "bitbucket-oauth": {},
    "github-oauth": {},
    "gitlab-oauth": {},
    "http-basic": {
        "repo.magento.com": {
            "username":"<public key>",
            "password":"<private key>"
        }
    }
}

保存成功之后就不用每次都去输入一长串的用户和密码了。
最后，执行 composer require bogardo/mailgun-magento2 命令自动安装成功。也不用每次输入密码了。

配置ssh key登录github

创建SSH Key

  1. 打开终端

  2. 输入以下命令 

    $ ssh-keygen -t rsa -b 4096 -C "your_email@example.com"

    根据输入的邮箱创建一个ssh key,填入邮箱之后会出现以下的提示信息。这里我们需要设置一个自定义名词。例如：id_rsa_work

    $ Enter a file in which to save the key (/Users/you/.ssh/id_rsa): [Press enter]

    执行完毕之后会生成 id_rsa_work 和 id_rsa_work.pub 两个文件。

添加生成的SSH Key到ssh-agent中

执行以下命令添加ssh key到到 ssh-agent 中。

$ ssh-add ~/.ssh/id_rsa_work

这里，id_rsa_work就是上面自定义的证书名词。

配置ssh

打开 ~/.ssh/config 文件(如果没有则创建一个)。编辑配置文件，使得能够自动识别该用哪个证书。注意缩进

Host work
    HostName github.com
    User git
    IdentityFile ~/.ssh/id_rsa

Host github.com
    HostName github.com
    User git
    IdentityFile ~/.ssh/id_rsa_person

部署到 github 上

  1. 点击顶部的 Setting 链接。
  2. 在账户左边的侧边,点击 SSH and GPG keys
  3. 点击 New SSH key.这里别点错了。
  4. 给 SSH KEY 取个名字.然后,复制文件 id_rsa_work.pub 的内容到输入框，点击add ssh key保存就好了。

添加源

最后添加源就可以了。

git remote add origin git@github.com:gavin9517/gavin9517.github.com.git
